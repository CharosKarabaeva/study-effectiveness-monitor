# Техническое задание

## Назначение системы
Разработка веб-сервиса для мониторинга, анализа и прогнозирования учебной эффективности
обучающегося на основе психоэмоциональных показателей и уровня продуктивности.

Система предназначена для регулярного ввода данных об учебных днях, их анализа,
визуализации и формирования рекомендаций по оптимизации учебного процесса.

## Роли пользователей

### Обучающийся (основной пользователь)
- вводит данные об учебных днях;
- просматривает статистику и графики;
- анализирует влияние настроения и усталости на продуктивность;
- получает рекомендации системы.

### Администратор системы
- управляет данными через административную панель Django;
- просматривает и редактирует записи пользователей.

## Функциональные требования

Система должна обеспечивать:
- добавление учебных дней с указанием даты, настроения, усталости, продуктивности и комментария;
- хранение данных в базе данных;
- фильтрацию учебных дней по выбранному диапазону дат;
- автоматический расчёт средних показателей учебной активности;
- визуализацию динамики продуктивности, настроения и усталости;
- анализ влияния психоэмоциональных факторов на продуктивность;
- классификацию учебных дней по уровню эффективности;
- формирование текстовых рекомендаций на основе накопленных данных;
- управление данными через административную панель.

## Структура данных

### Модель StudyDay
Представляет отдельный учебный день.

Поля:
- date — дата учебного дня;
- mood — уровень настроения (1–5);
- fatigue — уровень усталости (1–5);
- productivity — уровень продуктивности (1–5);
- comment — комментарий пользователя;
- created_at — дата и время создания записи.

Назначение: хранение первичных данных об учебной активности.

### Модель StudyMetric
Используется для агрегированного анализа учебной эффективности.

Поля:
- study_day — связь с учебным днём (ForeignKey);
- metric_type — тип метрики (настроение, усталость, продуктивность);
- value — числовое значение метрики.

Назначение: расширяемая модель для аналитических расчётов и статистики.

### Модель Recommendation
Хранит рекомендации, сформированные системой.

Поля:
- recommendation_type — тип рекомендации (информационная, предупреждение, положительная);
- text — текст рекомендации;
- created_at — дата формирования рекомендации.

Назначение: хранение и отображение рекомендаций пользователю.

## Архитектура системы
Система реализована на основе фреймворка Django с использованием архитектуры MTV
(Model–Template–View):

- Models — описание структуры данных и бизнес-логики;
- Views — обработка запросов, агрегация данных и формирование рекомендаций;
- Templates — пользовательский интерфейс и визуализация данных.

## Используемые технологии
- Python
- Django
- SQLite
- HTML / CSS
- Bootstrap
- Chart.js

## Ожидаемый результат
Работающий веб-сервис, доступный через браузер, позволяющий пользователю вести учёт
учебной активности, анализировать данные и получать рекомендации по улучшению
учебной эффективности.
